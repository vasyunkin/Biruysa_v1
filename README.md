# 🚤 Цифровой штурман аэролодок (MVP)

> Внутренний документ команды разработки
> Хакатон — MVP Offline-First навигационной системы

---

# Цель MVP

Разработать **offline-first веб-приложение**, которое строит маршрут по заранее подготовленному графу местности и рассчитывает:

* маршрут между двумя точками;
* время прохождения;
* расход топлива;
* остаток топлива;
* запас хода;
* предупреждения о рисках.

Важно понимать:

> **Главная ценность проекта — не карта, а навигационный движок (Routing Engine).**

Вся архитектура строится вокруг того, что в будущем вместо нашего JSON будут использоваться реальные данные контроллеров и датчиков аэролодки.

# Вес ребер

Это самое важное место всей системы.

Для каждого режима используется собственный вес.

---

## Быстрый

```
weight = time
```

где

```
time = distance / speed
```

---

## Экономичный

```
weight = fuel
```

где

```
fuel =
distance
× base
× surface_coef
× load_coef
× mode_coef
```

---

## Кратчайший

```
weight = distance
```

---

## Безопасный

```
weight =
distance
+
risk * коэффициент
```

или

```
weight = risk
```

если нужно максимально обходить опасные зоны.

---

# Что НЕ делать

❌ Не писать собственный алгоритм графов.

Использовать

```
NetworkX
```

---

❌ Не смешивать расчеты с API.

API должно только вызывать сервисы.

---

❌ Не хранить формулы внутри алгоритма.

Все коэффициенты должны находиться

```
weight_calculator.py
```

---

❌ Не привязывать алгоритмы к JSON.

Routing Engine должен получать уже готовый граф.

Тогда в будущем вместо JSON можно будет использовать реальные данные с аэролодки без переписывания логики.

# Минимальный результат MVP

К окончанию хакатона система должна уметь:

* ✅ загрузить граф;
* ✅ построить маршрут между двумя точками;
* ✅ учитывать выбранный режим движения;
* ✅ рассчитать время;
* ✅ рассчитать расход топлива;
* ✅ определить остаток топлива;
* ✅ вывести предупреждения;
* ✅ сравнить несколько вариантов маршрута;
* ✅ визуализировать найденный путь на графе.

Если все эти пункты выполнены, MVP полностью закрывает первые три сценария ТЗ и демонстрирует масштабируемую архитектуру, пригодную для дальнейшей интеграции с реальными навигационными системами аэролодок.
